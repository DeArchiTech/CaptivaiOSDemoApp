applyFilters:parameters:

Apply the set of filters, in the given order, to the current image. 

+(void)applyFilters:(NSArray *)filterNames parameters:(NSDictionary *)parameters;
Parameters
filterNames

Array of filter names (NSString). This can be any combination of:

CMSFilterAdaptiveBinary
This applies a best effort adaptive binary filter to the image. In the parameters argument, you can use the CMSFilterParamAdaptiveBinaryForce key as follows: YES - Forces the darkest pixels to be set to black, even when a comparison with the surrounding pixels would set them to white; NO (Default) - Uses the default behavior of the CMSFilterAdaptiveBinary filter. In the 2.0 release, this filter was updated to improve the handling of light text on dark backgrounds. To control the aggressiveness of light-on-dark handling, set a value for the CMSFilterParamAdaptiveBinaryBlackness key in the parameters argument. Valid values are 1-20 where lower values are more conservative about white text on lighter backgrounds; the recommended value is the default of 6. To retain the behavior from releases before 2.0, set this value to 2.
CMSFilterGrayscale
This applies grayscale filter to the image.
CMSFilterPerspective
This applies a best effort deskew filter to address 2D/3D skew issues and minor rotation issues. An optional argument, CMSFilterParamPerspectivePoints, can be used to supply four points ordered top-left, top-right, bottom-right, and bottom-left to define the perspective correction. This is an NSArray of CGPoint structures stored in NSValue objects.
CMSFilterCrop
This applies a crop to the image. A rectangle to crop can be provided in the parameters argument under the key of CMSFilterParamCropRectangle as a CGRect stored inside a NSValue object. Automatic cropping will be done if the rectangle is not provided. Padding for automatic cropping can been modified by the CMSFilterParamCropPadding float parameter. A value of 0.02 represents 2% padding.
CMSFilterResize
This resizes the image. The width and height need to be provided in the parameters argument as integers under the keys of CMSFilterParamResizeWidth and CMSFilterParamResizeHeight respectively.
CMSFilterRotation
This rotates the image. The rotation value needs to be provided in the parameters argument as an integer under the key of CMSFilterParamRotationDegree with any of the following acceptable values: 90, 180, 270. These values are clockwise rotation values.
CMSFilterBrightness
This changes the brightness of the image. The scaling factor needs to be provided in the parameters argument as an integer under the key of CMSFilterParamBrightnessScale with a value in the range -255 to 255. -255 produces the darkest image, 255 produces the lightest image, and 0 does not change the brightness.
CMSFilterContrast
This changes the contrast of the image. The scaling factor needs to be provided in the parameters argument as an integer under the key of CMSFilterParamContrastScale with a value in the range -255 to 255. -255 produces the lowest contrast, 255 produces the highest contrast, and 0 does not change the contrast.
CMSFilterRemoveNoise
This removes noise from the image. The size of the noise to remove needs to be provided in the parameters argument as an integer under the key of CMSFilterNoiseSize with a value greater than 0.
parameters

Filter specific parameters.

Discussion

The filters will be applied in the order they exist in the filterNames argument. A filter can only appear once in the list. As the (client-side) image filters are intended for processing images of captured documents, they are not applied if the current image is smaller than 100 pixels on either dimension. When this occurs, the source image is not modified and no error is returned. The crop and resize filter can be used to generate images smaller than 100 pixels, such as those used for display, but those images cannot be processed further.

autoCropRectWithThreshold:padding:

Calculate the edges for a given image and return the rectangle. 

+(CGRect)autoCropRectWithThreshold:(double)threshold padding:(double)padding;
Parameters
threshold

The percentage (0.12 = 12%) of a row or column which must be black for it to be considered an edge. If the threshold is less than or equal to 0 or greater than 1 then it defaults to 0.01

enableUndoImage:

Toggles an undo buffer capable of storing one change. 

+(void)enableUndoImage:(BOOL)undo;
Parameters
undo

YES to enable; NO to disable.

Discussion

The undo buffer stores one image used by undoImageChanges to restore the previous image. Disabling wll also clear the undo buffer and associated memory. This functionality could be used to implement a slider for a brightness control. The undo buffer can be freed with clearUndoImage. These operations can be restored by the undo buffer:

  • applyFilters:parameters:

  • loadFromBytes:error:

  • loadFromFile:error:

  • showQuadrilateralCrop:parameters:

measureQuality:

Measures the quality of the current image. 

+(int)measureQuality:(NSDictionary *)assessments;
Parameters
assessments

Dictionary with assesment names (NSString) as the key and weightings (NSNumber) as the value. The names can be any combination of:

CMSMeasureGlare
This determines the significance of the glare spot in the image, if any. Smaller values indicate more glare. Binary images and images without glare return 100.
CMSMeasureQuadrilateral
This determines whether there a quadrilateral detected in the image and if so, its size. Larger values indicate a larger quadrilateral. Images with no detected quadrilateral return 0.
CMSMeasurePerspective
This determines whether there a quadrilateral detected in the image and if so, its distortion. Larger values corners that are more square. Images with no detected quadrilateral return 0.

The assessments to use and their weights.

Discussion

The assessments listed in the assessments argument will be measured, each returning a value 0-100. These are then combined into a single weighted average to produce a value in the range 0-100. The weightings are normalized so that, for example, weightings of 4 and 1 are the same as weightings of 80 and 20. Any weighting outside the range 1-100 will be ignored. If the assessments argument is empty or contains only invalid values, the return value is 100.

saveToFile:encoding:parameters:error:

Call to save the image to a file. 

+(BOOL)saveToFile:(NSString *)filePath encoding:(NSString *)encoding 
        parameters:(NSDictionary *)parameters error:(NSError **)error;
Parameters
filePath

The full path to the file to which to save the image. May be nil if CMSSaveToGallery is YES.

encoding

Must be one of the following: CMSSaveJpg, CMSSavePng, or CMSSaveTiff.

parameters

The encoding parameters depend on the encoding:

Key=CMSSaveDpiX
Set the horizontal resolution in dots per inch in the metadata. If both resolution keys are not provided and the aspect ratio is between 1.2 and 1.5 then the resolution will be calculated from a letter sized page (8.5" x 11"). 200 will be used if the aspect ratio is out of that range. If only one key is provided then the other key is set to that value.
Key=CMSSaveDpiY
Set the vertical resolution in dots per inch in the metadata. If both resolution keys are not provided and the aspect ratio is between 1.2 and 1.5 then the resolution will be calculated from a letter sized page (8.5" x 11"). 200 will be used if the aspect ratio is out of that range. If only one key is provided then the other key is set to that value.
Key=CMSSaveJpgQuality
This is an integer that set the quality factor for compression. Acceptable values are 0 (poor quality but maximum compression) to 100 (Best quality but little or no compression). If not provided then 95 will be used.
Key=CMSSaveToGallery
A BOOL that indicates whether the image is to be saved to the gallery rather than a file. If set to YES, the filePath parameter is ignored.
Key=CMSSaveToGalleryCompletionSelector
A selector that indicates the callback to execute when the image is saved to the gallery rather than a file. The value is ignored unless CMSSaveToGalllery is set to YES. If set, the value must be an NSValue wrapping a SEL where the method signature is that of the selector for the iOS method UIImageWriteToSavedPhotosAlbum.
Key=CMSSaveToGalleryCompletionTarget
The id of an object whose selector is called if CMSSaveToGalleryCompletionSelector is set and CMSSaveToGalllery is YES. The value is ignored otherwise.
Key=CMSSaveToGalleryContext
The context passed to the selector if CMSSaveToGalleryCompletionSelector is set and CMSSaveToGalllery is YES. The value is ignored otherwise. If set, the value must be an NSValue wrapping a void*
error

The error object for the operation. If the function is used to save the image to the gallery, this return value is that from launching the save operation; if the operation fails, its error is passed to the completion selector.

Return Value

NO if not successful. The error object will contain further information.

Discussion

The encoding could be one of JPG, PNG, or TIFF. TIFF encoding only supports binary images.

+(BOOL)takePicture:(id<CMSImageDelegate>)callback parameters:(NSDictionary*)parameters 
        error:(NSError**)error;
Parameters
callback

The callback that will be called when a picture is taken. Mandatory (must not be nil).

parameters

Parameters that control the capture. The following key=value parameters are supported:

Key=CMSTakePictureContext
A UIViewController object from which takePicture() was invoked. takePicture() presents its UIViewController using the CMSTakePictureContext object, and then transitions back to this object when the picture is taken or cancelled. This parameter is mandatory and must not be nil.
Key=CMSTakePictureEdgeLabel
The edge label displays the provided text on the camera overlay on the long edge side of the phone. This is useful for displaying some instructional information to the user. If not provided, then nothing will be displayed. The text will be truncated to 150 characters. The text will remain on screen until the capture button is selected.
Key=CMSTakePictureCenterLabel
The center label displays the provided text in the middle of the screen. If not provided, then nothing will be displayed. The text will be truncated to 150 characters. The text will remain on screen until the capture button is selected.
Key=CMSTakePictureCaptureLabel
The capture label displays the provided text in the middle of the screen. If not provided, then nothing will be displayed. The text will be truncated to 150 characters. The text is displayed after the user selects the capture button and remains until the picture is taken.
Key=CMSTakePictureSensors
Provides control over the sensors that will be displayed and used for determining the optimal picture-taking conditions (see the description for CMSTakePictureOptimalConditions. If not provided, the default is "FM", which means all physical sensors will be displayed. This can be any combination of the following characters:

  • 'F' = Focus sensor.

  • 'M' = Motion sensor.

  • 'Q' = Quality sensor.

Key=CMSTakePictureMotionSensitivity
The amount of motion that can be tolerated by the motion sensor. A 0.0 float value indicates that absolutely no motion can be tolerated while 10.0 indicates that a large amount of motion can be tolerated. The default is 0.30.
Key=CMSTakePictureQualityMeasures
A NSDictionary object to enable image quality detection. See assessments in measureQuality:. Not meeting the quality level displays the quality sensor icon.
Key=CMSTakePictureCaptureTimeout
A positive integer representing the maximum number of milliseconds to wait for optimal conditions before taking the picture. See the description for CMSTakePictureOptimalConditions. A value of 0 indicates no wait. The default value is 0.
Key=CMSTakePictureCaptureDelay
A positive integer representing the number of milliseconds to wait from the time that the user selects the capture button to when the picture is taken. The default value is 500.
Key=CMSTakePictureOptimalConditions
If true, then the sensors as indicated in CMSTakePictureSensors and their corresponding thresholds (as specified in CMSTakePictureMotionSensitivity) must be satisfied before the user is allowed to take a picture. If focus is one of the sensors indicated in CMSTakePictureSensors, then the picture must be in focus. Furthermore, if CMSTakePictureCaptureTimeout has been exceeded, then onPictureCanceled: is called. The default value is NO.
Key=CMSTakePictureGuidelines
Boolean value to toggle display of guidelines. The default value is NO.
Key=CMSTakePictureCancelButton
Boolean value to toggle display of cancel button. The default value is NO.
Key=CMSTakePictureTorchButton
Boolean value to toggle display of torch button. The default value is NO.
Key=CMSTakePictureTorch
Boolean value to toggle torch mode. The default value is NO.
Key=CMSTakePictureImmediately
Boolean value to take a picture immediately without requiring a button click. The default value is NO.
Key=CMSTakePictureCrop
NSValue object holding a CGSize defining the dimension guides which the image will be cropped to. A special value, CMSTakePictureCropVisible, will be used to trim the image to what is visible in the current view.
Key=CMSTakePictureCropWarning
An UIColor object defining the warning color of the dimension guide. The default value is red.
Key=CMSTakePictureCropOk
An UIColor object defining the OK color of the dimension guide. The default value is green.
Key=CMSTakePictureCropProcessing
An UIColor object defining the processing color of the dimension guide. The default value is yellow.
Key=CMSTakePictureCaptureWindow
A class instance derived from CMSCaptureWindow to replace the default camera overlay.
Key=CMSTakePictureReturnMode
The actions to take after the picture is taken. The value can be a bitwise-or combination of the following. If omitted, the value CMSTakePictureReturnCallback is used.

  • CMSTakePictureReturnCallback - return the image through the callback.

  • CMSTakePictureReturnFile - save the image to the directory identified by CMSTakePicureSaveTo. The full path to the file is returned through the callback.

  • CMSTakePictureReturnLoad - load the image into the processing buffer.

Key=CMSTakePictureSaveTo
The path to the directory where taken pictures will be saved if the return mode includes CMSTakePictureReturnFile.
