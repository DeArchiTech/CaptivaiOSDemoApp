areAllSensorsValid

-(BOOL) areAllSensorsValid;
Return Value

YES if all sensors are valid; NO otherwise.

Discussion

The quality sensor is only measured after a candidate picture has been taken. During the idle state and while waiting, the return value for this method is based only on the other sensors.

cropGuidelineForPreview

-(CGRect) cropGuidelineForPreview;
Return Value

A rectangle in screen coordinates; the return value is empty if crop guidelines are not in use.

Discussion

This method only returns a valid rectangle during or after the first execution of the viewWillLayoutSubviews method. Custom UI can position a view at the location described by this rectangle to indicate to the user where the document should be positioned.

onCancelPicture:

-(void) onCancelPicture:(int)reason;
Parameters
reason

The reason capture is cancelled. The reasons and their meanings are:

  • CMSCancelReasonButton = Cancel button was pressed.

  • CMSCancelReasonTimeout = Optimal condition requirements were not met.

  • CMSCancelReasonError = Error occurred during image capturing process.

  • CMSCancelReasonPermission = No permission to access the camera.

  • CMSCancelReasonBackground = Application entered background status.

Discussion

Called when the picture taking process is being cancelled.

onSensorChange:valid:data:

-(void) onSensorChange:(NSInteger)sensor valid:(BOOL)valid 
        data:(NSObject*)data;
Parameters
sensor

The ID of the changed sensor. Supported sensors are:

CMSSensorMovement
The device has moved. The data is a CMDeviceMotion object.
CMSSensorFocus
Focus has changed. No data is returned.
CMSSensorQuality
The quality of the image has changed. The data is an NSDictionary as described in the discussion.
valid

YES if the sensor meets the thresholds; NO otherwise

data

A data object that is appropriate for the sensor.

Discussion

When the quality sensor changes, the returned data dictionary contains the following entries:

CMSQualityCorners
NSArray of NSValue objects wrapping CGPoint.

onShowCaptureMode:

-(void) onShowCaptureMode:(int)mode;
Parameters
mode

The new capture mode. The modes and their meanings are:

CMSCaptureModeIdle
The application has just launched the capture UI and the user has not yet clicked the capture button.
CMSCaptureModeCapturing
The application is capturing images.
Discussion

Called when the SDK changes the capture mode.
